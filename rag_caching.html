RAG Caching —
Types, Layers, and Strategies

Retrieval-Augmented Generation · FastAPI + Python · July 2026

RAG (Retrieval-Augmented Generation) works as a pipeline — document → parse → chunk → embed → vector store → retrieve → LLM prompt → response. Every step has a cost: API dollars, CPU time, disk I/O, latency. The same document gets chunked every restart. The same query hits the vector store repeatedly. The same context block gets sent to Claude on every chat message.

Caching stores the output of each step so subsequent requests skip it. The right cache at the right layer can reduce latency by 10× and cost by 90%.

Layer 1

Document Preprocessing Cache

Caches the output of parsing and chunking raw documents — PDFs, Excel, Word files. Parsing a 50-page PDF with pdfminer takes 2–5 seconds per file. Re-chunking 100 documents on every restart wastes minutes.

What to store

# key = SHA-256 of the raw file bytes
SHA256(file_bytes) → {
  "chunks": ["chunk 1 text...", "chunk 2 text...", ...],
  "metadata": { "pages": 50, "parsed_at": "2026-07-13" }
}

Code example — diskcache

import diskcache, hashlib

_cache = diskcache.Cache("data/.chunk_cache")

def chunk_with_cache(file_bytes: bytes, chunk_fn) -> list[str]:
    key = hashlib.sha256(file_bytes).hexdigest()
    if key in _cache:
        return _cache[key]
    chunks = chunk_fn(file_bytes)
    _cache.set(key, chunks, expire=7 * 24 * 3600)   # 7 days
    return chunks

Layer 2

Embedding Cache

The vector representation of each text chunk — "chunk text..." → [0.23, 0.11, -0.44, ...]. This is the most expensive repeated operation in RAG. Calling the embedding API costs time and money on every re-index.

Without embedding cache
~$2.00
per 100K chunks re-indexed
(OpenAI ada-002, 200 tokens/chunk)
With embedding cache
$0.00
for every subsequent re-index run
unless the model changes

LangChain built-in (drop-in, one change)

from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore
from langchain_openai import OpenAIEmbeddings

store = LocalFileStore("data/.embedding_cache")
base  = OpenAIEmbeddings(model="text-embedding-ada-002")

# Drop-in replacement — caches automatically by text hash + namespace
cached_embedder = CacheBackedEmbeddings.from_bytes_store(
    base, store, namespace=base.model
)

Custom Redis cache

import redis, hashlib, numpy as np

r = redis.Redis()

def embed_cached(text: str, embed_fn, model="ada-002") -> list[float]:
    key = f"emb:{hashlib.sha256(f'{model}:{text}'.encode()).hexdigest()}"
    cached = r.get(key)
    if cached:
        return np.frombuffer(cached, dtype=np.float32).tolist()
    vec = embed_fn(text)
    r.set(key, np.array(vec, dtype=np.float32).tobytes(), ex=90 * 86400)
    return vec

TTL: 30–90 days. Embeddings are deterministic — same text + same model always produces the same vector. Only invalidate if you switch embedding models.

Layer 3

Vector Index Cache (In-Memory)

Keeps the loaded FAISS or ChromaDB index in RAM as a process-level singleton. Loading a FAISS index with 100K vectors from disk takes 500 ms – 2 s per request — unacceptable for interactive search.

# dependencies.py — consistent with existing repo singleton pattern
import faiss, json
from pathlib import Path

_index: faiss.Index | None = None
_chunks: list[str] = []

def get_vector_index() -> tuple[faiss.Index, list[str]]:
    global _index, _chunks
    if _index is None:
        _index  = faiss.read_index(str(Path("data/faiss.index")))
        _chunks = json.loads(Path("data/chunks.json").read_text())
    return _index, _chunks

def invalidate_index():
    global _index, _chunks
    _index, _chunks = None, []   # next request reloads from disk

Layer 4

Retrieval Result Cache

Caches the top-k chunks returned by a vector similarity search for a given query. Dashboards refreshing, multiple users asking the same question — the ANN search runs identically each time.

from cachetools import TTLCache
import hashlib, json

_retrieval: TTLCache = TTLCache(maxsize=500, ttl=120)   # 2-minute TTL

def retrieve_cached(query: str, index, k: int = 5) -> list[str]:
    key = hashlib.sha256(
        json.dumps({"q": query, "k": k}, sort_keys=True).encode()
    ).hexdigest()
    if key in _retrieval:
        return _retrieval[key]
    results = index.search(query, k=k)
    _retrieval[key] = results
    return results

Hit rate: typically 15–40% in multi-user deployments. TTL: keep short (1–5 min) — bid data changes throughout the day.

Layer 5

Semantic Query Cache

Matches semantically similar past queries — not just exact hash matches. "What is the response rate for BID-089?" and "How many suppliers responded to BID-089?" should return the same cached answer.

How it works

1. Embed the incoming query        → q_vec
2. Search a small "query cache"     → past (query_vec, response) pairs
3. If cosine_similarity ≥ 0.95    → return cached response
4. Otherwise                        → run full RAG pipeline, store result

Code example

import numpy as np
from dataclasses import dataclass

@dataclass
class CachedQA:
    query_vec: list[float]
    response:  str

_sem_cache: list[CachedQA] = []
THRESHOLD = 0.95

def _cos(a, b) -> float:
    a, b = np.array(a), np.array(b)
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))

def sem_lookup(query: str, embed_fn) -> str | None:
    q_vec = embed_fn(query)
    for entry in _sem_cache:
        if _cos(q_vec, entry.query_vec) >= THRESHOLD:
            return entry.response   # cache hit
    return None

def sem_store(query: str, response: str, embed_fn) -> None:
    _sem_cache.append(CachedQA(embed_fn(query), response))
Similarity thresholdBehaviourRisk
0.99Near-identical phrasing onlyLow — very conservative
0.95Catches paraphrases, common synonymsRecommended
0.90Aggressive fuzzy matchingHigh — false positives for different intents

Do NOT use semantic cache for queries containing specific IDs ("BID-089" vs "BID-090") or real-time data where stale answers are harmful. Small semantic differences can mean very different correct answers.

Layer 6

LLM Response Cache (Exact Match)

Caches the final LLM response for an exact prompt hash. Useful for scheduled reports, health-check bots, or during development to avoid burning API quota.

LangChain built-in (one line at startup)

from langchain.globals import set_llm_cache
from langchain.cache import SQLiteCache, RedisCache, InMemoryCache
import redis

# Choose one:
set_llm_cache(SQLiteCache(".langchain_cache.db"))     # disk, persists restarts
set_llm_cache(InMemoryCache())                           # process lifetime only
set_llm_cache(RedisCache(redis_=redis.Redis(), ttl=300)) # 5-minute TTL

# All subsequent LangChain LLM calls are now cached automatically — no other changes
SituationUse LLM cache?
Scheduled hourly reportYes — same query every run
Development / testingYes — avoid API cost during iteration
Live conversational chatNo — context changes per message
Queries with real-time bid dataNo — stale response worse than no cache

Integration into routers/ai_chat.py

# Split static context (cached) from the live user turn
async def chat(body: ChatRequest, llm=Depends(get_llm_service)):
    context = await retrieve_bid_context(body.bid_id, body.messages[-1].content)

    response = await llm.chat_with_cache(
        static_context=context,      # marked cache_control — charged once
        messages=body.messages,      # live conversation — never cached
    )
    return {"reply": response}

Cache TTL: 5 minutes from last use. Each read resets the window — as long as users send one message every <5 minutes, the cache stays warm for the entire session.

Layer 8

API Route Cache

Caches the full HTTP response at the FastAPI route level — before any RAG or LLM work begins. Dashboard endpoints called on every page refresh by every user are ideal candidates.

fastapi-cache2 (recommended)

Lightweight in-memory alternative (no Redis)

from cachetools import TTLCache
_route_cache: TTLCache = TTLCache(maxsize=100, ttl=60)

def route_cache(ttl: int = 60):
    def deco(fn):
        import functools
        @functools.wraps(fn)
        async def wrapper(*args, **kwargs):
            key = str(kwargs)
            if key in _route_cache:
                return _route_cache[key]
            result = await fn(*args, **kwargs)
            _route_cache[key] = result
            return result
        return wrapper
    return deco

_ver = 0

def cache_key(q: str) -> str:
    return f"v{_ver}:{hashlib.sha256(q.encode()).hexdigest()}"

def invalidate_all():
    global _ver
