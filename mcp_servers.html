01

What is MCP?

MCP (Model Context Protocol) is an open standard created by Anthropic (released November 2024) that defines a universal way for AI models to connect to external tools, data sources, and services — without writing a custom integration for each one.

The USB Analogy

Before USB
Every device had a proprietary connector. Printers, cameras, keyboards — each needed its own plug and driver.
→
After USB
One universal standard. Any device plugs into any computer. No custom drivers, no proprietary connectors.

MCP is USB for AI. You write one MCP server for your tool. Any MCP-compatible AI — Claude Desktop, Claude Code, Cursor, VS Code Copilot — can use it automatically, with no custom glue code.

The Problem MCP Solves

Without MCP, if you want Claude to reason about your data, you must fetch everything, serialize it into a text block, inject it into the prompt, and hope it fits in the context window. That works for small datasets — but it breaks down fast.

With MCP, Claude asks for what it needs. Your server responds with live, precise data. The prompt stays small. The data stays current.

03

The Three Things MCP Servers Expose

🔧 Tools
Functions the AI can call. The AI decides when to call them — you just define what they do.

Like API endpoints, but the AI is the client.
📄 Resources
Data the AI can read on demand. Identified by a URI (like a file path or URL).

For larger, less dynamic data the AI reads passively.
💬 Prompts
Reusable prompt templates stored on the server side.

Standard instructions your team wants every AI session to start with.
# Tools — the AI calls these on-demand
@mcp.tool()
async def get_bid(bid_id: str) -> dict:
    """Get full bid details by ID."""
    return await bids_repo.find_one(lambda b: b["id"] == bid_id)

# Resources — the AI reads these like a file
@mcp.resource("bids://active")
async def active_bids() -> str:
    """All bids in Active or Solicitation status."""
    bids = await bids_repo.load()
    return str([b for b in bids if b["status"] in ("Active", "Solicitation")])

How to Build and Run an MCP Server

1
Install the SDK
Inside backend_py/:
pip install mcp
2
Create backend_py/mcp_server.py
from mcp.server.fastmcp import FastMCP
from repositories.bids_repo import BidsRepository
from repositories.suppliers_repo import SuppliersRepository

mcp = FastMCP(
    name="sysco-rfpaiq",
    instructions="Tools for the Sysco RFP AIQ bid management platform."
)

# ── Bid tools ───────────────────────────────────────────────
@mcp.tool()
async def get_bid(bid_id: str) -> dict:
    """Get full details for a bid by its ID."""
    repo = BidsRepository("data/bids.json")
    result = await repo.find_one(lambda b: b.get("id") == bid_id)
    return result or {"error": f"Bid {bid_id!r} not found"}

@mcp.tool()
async def list_bids(
    segment: str = "",
    status: str = "",
) -> list[dict]:
    """List bids, optionally filtered by segment or status."""
    repo = BidsRepository("data/bids.json")
    bids = await repo.load()
    if segment: bids = [b for b in bids if b.get("segment") == segment]
    if status:  bids = [b for b in bids if b.get("status")  == status]
    return [{"id": b["id"], "customer": b.get("customer"),
             "status": b.get("status"), "customerDue": b.get("customerDue")} for b in bids]

# ── Supplier tools ───────────────────────────────────────────
@mcp.tool()
async def search_suppliers(query: str, category: str = "") -> list[dict]:
    """Search suppliers by name or category."""
    repo = SuppliersRepository("data/suppliers.json")
    suppliers = await repo.load()
    term = query.lower()
    results = [s for s in suppliers
               if term in s.get("name", "").lower()
               or term in s.get("category", "").lower()]
    if category: results = [s for s in results if s.get("category") == category]
    return results[:10]

if __name__ == "__main__":
    mcp.run()  # stdio transport — Claude connects to this process
3
Connect to Claude Code
Add to .claude/settings.json in your project root:
{

Upgrading the Existing AI Chat Router

The biggest win: replace _build_system_prompt() in routers/ai_chat.py with LangChain tool-calling. LangChain is already used in this project via services/llm/factory.py.

# Bind tools to the model — it decides when to call each one
llm_with_tools = llm.bind_tools([get_bid_tool, list_bids_tool, search_suppliers_tool])

# The system prompt is now tiny — no pre-fetched data needed
SYSTEM = "You are the Sysco RFP AIQ assistant. Use the tools to answer questions about bids and suppliers."

result = await llm_with_tools.ainvoke([

How it works at runtime: The model reads the user's question, decides which tool to call (e.g. get_bid_tool("BID-089")), receives the result, and continues reasoning. This loop happens automatically — you just define what each tool does.

Production: HTTP + SSE Transport

Once deployed to Azure or AWS, mount the MCP server onto the existing FastAPI app so any Claude client can reach it via HTTPS:

# backend_py/main.py — add to existing FastAPI app
from mcp.server.fastmcp import FastMCP
from mcp.server.fastmcp.transport.sse import add_sse_routes

mcp = FastMCP("sysco-rfpaiq")
# ... register tools ...

# Mount at /api/mcp alongside existing routes
add_sse_routes(app, mcp, path="/api/mcp")

Claude Code connects via the URL:

{
  "mcpServers": {
    "sysco-rfpaiq": {
      "url": "https://rfpaiq.sysco.com/api/mcp"
    }
  }
}
